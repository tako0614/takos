# Takos Deploy v2 v1.0 Implementation Checklist

Use this checklist to verify that an implementation conforms to the v1.0 Core semantics.

- [ ] Core contract uses MUST / SHOULD / MAY consistently.
- [ ] TypeScript model and JSON Schema match the normative record names.
- [ ] DescriptorResolution policy decisions are recorded for descriptor use.
- [ ] Descriptors are declarative and cannot hide executable provider logic.
- [ ] Provider capabilities constrain or reject contracts but do not reinterpret them.
- [ ] ActivationRecord invariants are covered by conformance tests.
- [ ] GroupActivationPointer selects the current immutable ActivationRecord.
- [ ] Route-specific assignment, if implemented, is stored in ActivationRecord-owned state.
- [ ] Publication does not imply injection.
- [ ] Resource credentials are not publication outputs by default.
- [ ] Provider observation is not canonical state.
- [ ] Rollback uses retained artifacts and retained descriptor digests.
- [ ] Current Takos migration plan is mapped to the Core objects.
- [ ] Official descriptor set is available before Plan.
- [ ] Condition reason catalog is used consistently by API, CLI, UI, and controllers.
