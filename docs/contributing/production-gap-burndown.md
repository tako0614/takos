# Production Gap Burndown

This is the concise burndown for the current Takos PaaS current surface. It
records the strongest state currently present in the `takos/paas` product root
and keeps external self-host/cloud proof separate from the kernel release gate.
Run proof commands from `/home/tako/Desktop/takos/takos/paas`.

## Current validation baseline

- Deployment-centric kernel surface is implemented in
  `apps/paas/src/domains/deploy/deployment_service.ts`.
- Public PaaS routes expose the Deployment / GroupHead / ProviderObservation
  surface and omit removed public plan/apply/snapshot paths.
- `scripts/release-gate.ts` is the safe local gate. It runs type check, tests,
  lint, fmt check, docs lint, docs build, process-role and architecture
  validators, core validators, Helm validation, distribution manifest
  validation, distribution dry-run smoke, router config smoke, runtime-agent API
  smoke, release manifest generation, and PaaS in-process smoke.
- Official distribution manifests live in `deploy/distributions`. They are
  public templates, not live proof. Live self-host/cloud proof requires
  operator-owned infrastructure, credentials, and external plugin live tasks.

## Remaining high-priority gaps

- Live self-host/cloud proofs are not covered by local kernel gates. Keep them
  as operator-owned evidence attached to the distribution target.
- RepairPlan rematerialization and rollback-with-migration resource semantics
  remain backlog unless covered by a concrete test in a later turn.
- Resource feature planning still needs tests for required provider features,
  pgvector/native feature realization, and cross-contract rename safety.
- Canary side-effect safety still needs tests for queue DataContract policy,
  candidate-scoped egress, shadow traffic side-effect guards, and DB semantic
  write approvals.
- Managed projection health and ChangeSetPlan partial-failure semantics remain
  acceptance gaps.
