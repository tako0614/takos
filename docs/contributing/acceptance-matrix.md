not fully asserted. **Gap** means no current implemented test was found.

## AppInstallation lifecycle acceptance

Installable App Model (ROADMAP.md Phase 1.1-1.7) で追加される所有権 / install
ライフサイクルの acceptance entries。**current implementation status は Phase
1.1〜1.7 進捗に合わせて update する** こと (各 phase の merge 時に該当行の
Status / Implemented tests を更新)。

**この matrix と [acceptance-test-backlog.md](./acceptance-test-backlog.md)
の分担**: 本 matrix は **release gate 観点** で「閉じる必要のある acceptance

## Remaining high-priority gaps

- Plan/Apply revalidation: long migration lock lifetime and pre-Apply
  PreparedArtifact package-resolution validation.
- Rollback/repair: rollback-with-migration resource semantics and RepairPlan
  rematerialization. GroupHead pointer rollback and retained-history selection
  are covered.
- Canary side-effect safety: queue DataContract policy and hard-fail shadow
  traffic side-effect semantics.
- Publications/security: ChangeSetPlan partial failure semantics.
- Direct deploy image/source/bundle compilation, native env destination,
  Deployment enforcement, and manifest-managed mutation protection are covered.
- Retention edge cases are now covered for rollback-window workload revisions,
  prepared-artifact TTL expiry, and provider-package retention references.
- External self-host/cloud live proof is intentionally separate from the kernel
  matrix; distribution manifests and dry-run smoke are covered here, while real
  provider proof belongs to the Takosumi (`@takosumi/plugins`) live smoke tasks.
