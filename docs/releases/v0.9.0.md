# Takos v0.9.0 - Installable App Migration Announcement

Release date: 2026-05-12

## Summary

- Release candidate commit: `dae8319319d982204671f4bce0cb7263eeb1909e`
- Production deployment id: not deployed by this announcement release
- Release owner: Release owner / operator
- Operator: operator account plane, reference distribution `takosumi-cloud`
- Release window: announcement window for the Takos 1.x Installable App Model
  migration

This is the public announcement release for the migration away from Takos-owned
legacy account, billing, credential, and direct deploy surfaces. It starts the
documented migration-window evidence sequence. It does not close the ROADMAP
community-announcement gate by itself; deprecation and removal releases with
separate public evidence are still required.

## What Changed

- Takos is treated as an OIDC consumer of the operator account plane.
- App ownership, billing, grants, bindings, launch tokens, and runtime mode are
  handled through the AppInstallation ledger.
- AppInstallation-owned app lifecycle deploys use GitOps deploy intent and the
  Installable App Model.
- The Takosumi kernel `POST /v1/deployments` remains the canonical unmanaged
  deploy entry point and is not part of this Takos product legacy removal.

## Breaking Changes

- Breaking change present in this release: no.
- This announcement names future removal scope; it does not remove additional
  public routes by itself.

Affected legacy surfaces:

- Takos-owned OAuth/OIDC issuer endpoints: `/oauth/*`, `/api/public/v1/oauth/*`,
  `/api/oauth/*`, and OAuth provider discovery.
- Takos app-local credential and PAT issuance surfaces that conflict with
  Takosumi Accounts-issued bearer/PAT credentials.
- Takos app-owned billing owner surfaces that conflict with operator BillingPort
  ownership.
- Takos app public/proxy direct deploy routes that bypass GitOps deploy intent
  for AppInstallation-owned app lifecycle operations.
- Agent or managed-skill direct deploy affordances such as
  `group_deployment_snapshot_deploy_from_repo`.

Compatibility window:

- Announcement release: `v0.9.0`
- Deprecation release: not published yet; must include warning or audit
  evidence.
- Removal release: not published yet; must include deleted, `410`, or `404`
  route evidence and release-gate evidence.

## Migration Guide

Operator migration guide:
[`docs/operator/account-migration.md`](../operator/account-migration.md)

Required operator action:

- Configure the operator account plane as the OIDC issuer for Takos.
- Move account, credential, upstream IdP, billing, and AppInstallation lifecycle
  ownership to Takosumi Accounts or a contract-compatible operator distribution.
- Keep direct kernel deploy use on the unmanaged `POST /v1/deployments` path
  only; do not treat it as an AppInstallation-owned app lifecycle deploy.

Required user or integration action:

- Move automation from Takos app-local credentials or PATs to operator-issued
  bearer/PAT credentials.
- Stop depending on Takos-owned OAuth issuer URLs.
- Move AppInstallation-owned deploy automation to GitOps deploy intent.

## Rollback Plan

No production deployment is performed by this announcement release. If a
deployment later references this release and must be rolled back, use the
operator rollback SOP and keep the operator account-plane AppInstallation ledger
intact. Do not roll back by restoring Takos-owned OAuth issuer, billing, or PAT
ownership as current contract.
