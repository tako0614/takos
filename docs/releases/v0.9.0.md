# Takos v0.9.0 - Installable App No-User Clean Cut

Release date: 2026-05-13

## Summary

- Release candidate commit: the commit tagged `v0.9.0`
- Production deployment id: not deployed by this release
- Release owner: Release owner / operator
- Operator: operator account plane, reference distribution `takosumi-cloud`
- Release window: no-user clean cut for the Takos 1.x Installable App Model
  cleanup

This is the single clean release for the Takos-owned legacy account, billing,
credential, and AppInstallation-owned direct deploy cleanup. Takos has no
existing public users or external operators in this pre-GA line, so there is no
user data, integration, or tenant fleet to migrate. No compatibility migration
window is required.

## What Changed

- Takos remains a product running on Takosumi, not a platform/account-plane
  owner.
- Takos app stays an OIDC consumer of the operator account plane.
- App ownership, billing, grants, bindings, launch tokens, and runtime mode are
  handled through the AppInstallation ledger.
- AppInstallation-owned app lifecycle deploys use GitOps deploy intent and the
  Installable App Model.
- Takos-owned legacy account, billing, credential, and direct deploy surfaces
  are removed or verified absent from current product code.
- The Takosumi kernel `POST /v1/deployments` remains the canonical unmanaged
  deploy entry point and is not part of this Takos product cleanup.

## Breaking Changes

- Breaking change present in this release: yes, for pre-GA compatibility
  surfaces only.
- Affected users or integrations: none known; operator assertion is that Takos
  has no existing public users or external operators for this release line.
- Migration window: not required because there are no users to migrate.

Affected Takos product-owned legacy surfaces:

- Takos-owned OAuth/OIDC issuer endpoints: `/oauth/*`, `/api/public/v1/oauth/*`,
  `/api/oauth/*`, and OAuth provider discovery.
- Takos app-local credential and PAT issuance surfaces that conflict with
  operator account-plane issued bearer/PAT credentials.
- Takos app-owned billing owner surfaces that conflict with the operator
  account-plane BillingPort.
- Takos app public/proxy direct deploy routes that bypass GitOps deploy intent
  for AppInstallation-owned app lifecycle operations.
- Agent or managed-skill direct deploy affordances such as
  `group_deployment_snapshot_deploy_from_repo`.

## Migration Guide

No user migration is required for this no-user clean cut. The operator-facing
reference remains:
[`docs/operator/account-migration.md`](../operator/account-migration.md)

Required operator action for future deployments:

- Keep Takos configured as an OIDC consumer of the operator account plane.
- Keep account, credential, upstream IdP, billing, and AppInstallation lifecycle
  ownership in Takosumi Accounts or a contract-compatible operator distribution.
- Keep AppInstallation-owned deploy automation on GitOps deploy intent.
- Keep direct kernel deploy use on the unmanaged `POST /v1/deployments` path
  only.

## Rollback Plan

No production deployment is performed by this release. If a deployment later
references this release and must be rolled back, use the operator rollback SOP
and keep the operator account-plane AppInstallation ledger intact. Do not roll
back by restoring Takos-owned OAuth issuer, billing, PAT, or
AppInstallation-owned direct deploy ownership as current contract.
