# Takos v0.9.2 - Installable App Migration Removal

Release date: pending; publish no earlier than 2026-05-14 JST

## Summary

- Release candidate commit: the commit tagged `v0.9.2`
- Production deployment id: not deployed by this removal release
- Release owner: Release owner / operator
- Operator: operator account plane, reference distribution `takosumi-cloud`
- Release window: removal window for the Takos 1.x Installable App Model
  migration
- Status: release-note draft until the public GitHub release is published

This is the prepared removal release note for the Takos-owned legacy account,
billing, credential, and AppInstallation-owned direct deploy surfaces named in
`v0.9.0` and deprecated in `v0.9.1`. It must not be used as completion evidence
until a distinct public Takos `v0.9.2` release and matching takos-ecosystem
sign-off release are published on a project date later than `v0.9.1`.

## Removal Scope

The following Takos product-owned legacy surfaces are removed or verified absent
from current product code:

- Takos-owned OAuth/OIDC issuer endpoints: `/oauth/*`, `/api/public/v1/oauth/*`,
  `/api/oauth/*`, and OAuth provider discovery.
- Takos app-local credential and PAT issuance surfaces that conflict with
  operator account-plane issued bearer/PAT credentials.
- Takos app-owned billing owner surfaces that conflict with the operator
  account-plane BillingPort.
- Takos app public/proxy direct deploy routes that bypass GitOps deploy intent
  for AppInstallation-owned app lifecycle operations.
- Agent or managed-skill direct deploy affordances such as
  `group_deployment_snapshot_deploy_from_repo`.

The Takosumi kernel `POST /v1/deployments` remains the canonical unmanaged
deploy entry point and is not part of this Takos product legacy removal.

## Route Evidence

The removal release must include current command output showing the retired Takos
product-owned routes are deleted or return the expected `404` / `410` responses,
and that direct deploy affordances remain absent from current product code:

```sh
cd takos
deno task validate:legacy-removal

cd ..
rg -n "group_deployment_snapshot_deploy_from_repo|group_deployment_snapshot\\.deploy_from_repo|deploy_from_repo" \
  takos/app takos/agent -g '!node_modules' -g '!target'
```

The `rg` command must return no hits for current Takos product code. This scan
does not target takosumi kernel deploy APIs.

## Migration Guide

Operator migration guide:
[`docs/operator/account-migration.md`](../operator/account-migration.md)

Required operator action:

- Keep Takos configured as an OIDC consumer of the operator account plane.
- Keep account, credential, upstream IdP, billing, and AppInstallation lifecycle
  ownership in Takosumi Accounts or a contract-compatible operator distribution.
- Keep AppInstallation-owned deploy automation on GitOps deploy intent.
- Keep direct kernel deploy use on the unmanaged `POST /v1/deployments` path
  only.

Required user or integration action:

- Stop depending on Takos-owned OAuth issuer URLs.
- Move Takos app-local credentials or PAT automation to operator-issued
  bearer/PAT credentials.
- Move AppInstallation-owned deploy automation to GitOps deploy intent.

## Rollback Plan

No production deployment is performed by this removal release. If a deployment
later references this release and must be rolled back, use the operator rollback
SOP and keep the operator account-plane AppInstallation ledger intact. Do not
roll back by restoring Takos-owned OAuth issuer, billing, PAT, or
AppInstallation-owned direct deploy ownership as current contract.
